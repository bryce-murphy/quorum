# PMN-001 - Adversarial review is required ceremony for the verifier's trust boundary

**Status:** committed
**Cycle:** QRM-1 (Phase 1 vertical slice - L0 contracts + L1 verification kernel)
**Principles:** 1 (verify, don't trust), 7 (cross-family review), 10 (external evidence or it didn't happen), 11 (delete by default)

## Observation

Across QRM-1 the kernel passed same-family review - the Builder's self-check plus the Architect's line-by-line re-reads - and a full green test suite, yet four consecutive cross-family red-team rounds each found a defect that defeated the verifier's core promise. In order of discovery, the headline defects were: unclaimed changes riding a clean ledger (the diff-coverage gap), attacker-supplied `--base` delta carving, the `main`-absent carving edge case, and existence-only coverage being accepted at high tier. Every one of them sat on the verifier's coverage / base / tier trust boundary. Same-family review - including the Architect's own reads, which share the Builder's blind spots by construction (same model family) - passed several of them.

## Evidence class

External (principle 10): independent cross-model (GPT / Codex) review, repeated across four rounds, each with a reproduction. This is the strongest evidence tier short of an external adopter. It is empirical, not anecdotal.

## Discipline promoted

Any change touching the verifier's coverage / base / tier / trust-boundary logic requires a cross-family red-team pass as a required gate, not merely correctness review. In M3 this becomes enforceable: the Gate can require the red-team gate for T3 changes to enforcement machinery. Changes off that boundary (e.g. CLI presentation such as QRM-2.1) take review + Architect re-read only - over-ceremony is also a failure mode (principle 3).

## Sunset trigger (principle 11)

If, over a defined number of instrumented cycles on this surface, red-team passes stop finding defects that review missed, downgrade the requirement from a required gate to a spot-check. Do not keep ceremony that has stopped catching defects. This note retires itself the moment its own evidence dries up.

## Provenance

The QRM-1 state this note rests on was independently re-verified from `main`: commit log, clean build, full suite, and schema drift all confirmed by re-running them, not by trusting the hand-off. The defect classes above are recorded in `docs/handoffs/QRM-1-pr-body.md` and are covered by adversarial regression tests in `packages/kernel/test/`.
