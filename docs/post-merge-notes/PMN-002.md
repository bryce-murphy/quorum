# PMN-002 - Hand-back fidelity: the Builder's report is a claim, verified against git, not the summary

**Status:** committed
**Cycle:** QRM-2.1 (empty-delta decline) - incident; reaffirmed during M3 scoping
**Principles:** 1 (verify, don't trust), 5 (git is the source of truth), 3 (proportionate ceremony), 10 (evidence honesty), 11 (delete by default)

## Observation

During QRM-2.1 the Builder handed back reporting a committed branch; the change existed only in the worktree. The branch ref pointed at `main`, so the PR would have shipped an empty delta. A `git diff --stat` reads identically whether a change is committed or merely worktree-resident, so the diffstat pasted into the hand-back summary looked correct. Cross-family review reading the actual ref - not the summary - caught it. This is a distinct failure class from PMN-001: not a trust-boundary defect in the verifier, but a fidelity gap between an agent's self-report and git ground truth (principle 5).

## Evidence class

Internal / process (principle 10): a single witnessed incident inside the pipeline, caught by cross-family review. Weaker than PMN-001's repeated external red-team evidence. Stated honestly: it is a real, reproducible event (the ref pointed at `main`), but one occurrence on our own process - not an external adopter or cross-domain datum. Promote it as standing hand-back discipline, not as a hardened invariant.

## Discipline promoted

At every hand-back, before the Architect clears anything, require and verify directly against git - never against the pasted summary:
1. a non-empty `git diff --stat <base>...<branch>` that the Architect computes itself;
2. the branch HEAD sha, confirmed to match the remote ref after push;
3. the loop's terminal status (success vs. a turn-cap, budget-cap, or execution-error termination).

The Architect clears against the committed diff it computed, not the hand-back narrative.

## Termination-status clause (reasoned extension - not yet witnessed; principle 10)

The Builder runs as an agent loop whose final message is the model's narration, not a measurement. A loop can terminate on a turn cap, budget cap, or execution error mid-task, leaving a partial commit that the hand-back may present as complete. Diff-coverage catches *unclaimed* paths but not work that is internally consistent yet *functionally incomplete*. Therefore the hand-back must state the loop's terminal status, and the Architect treats any non-success termination as presumptively incomplete regardless of how clean the diff appears, pending explicit confirmation that the committed delta is the whole intended change. This clause is reasoned from the loop's result semantics, not from a witnessed incident; it is labeled as such and is the first sunset candidate below.

## Sunset trigger (principle 11)

If, across a defined number of instrumented cycles, the git-state checks and the termination-status check never diverge from the Builder's summary, downgrade from a required hand-back gate to a spot-check. The termination-status clause retires first if no non-success-termination incident is ever observed. Do not keep ceremony that has stopped catching defects.

## Provenance

The QRM-2.1 incident is described in the M3 Architect handoff (sections 3 and 7). The repo state this note rests on was independently re-verified from `main` @ `442f68b` - commit log, clean build, full suite (104 passed), schema drift (4/4) - by re-running them, not by trusting the hand-off.
